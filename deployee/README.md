# Deployee

Deployee is a WordPress-admin-only deployment workspace for this project.

The left navigation is generated by `ft_deployee_systems()` in `index.php`.
Add future tools beneath `deployee/systems/` and register their name,
description, icon, and URL in that function.

Open:

`http://localhost/floorstoday/deployee/`

## Recommended workflow

1. Save the local and CloudPanel URLs in Environment Settings.
2. Use **Create Backup** before major local changes.
3. Use **Build Server SQL** to export a serialization-safe database with local URLs converted to the server URL.
4. Push code with Git.
5. Pull the code in the CloudPanel site directory.
6. Upload/import the generated SQL on the server.
7. Flush WordPress and page caches.

## Pulling the live database locally

1. Export the server database as SQL.
2. Open local Deployee.
3. Choose the SQL file under **Import server database**.
4. Type `IMPORT DATABASE`.
5. Deployee creates a local safety backup, imports the SQL, and replaces the configured server URL with the local URL.

## Security

- WordPress administrator authentication is required.
- Every action uses a WordPress nonce.
- SQL backups and settings are excluded from Git.
- Direct web access to `deployee/storage` is denied by `.htaccess`.
- Database credentials remain in `wp-config.php`; Deployee does not save them.

On Nginx/CloudPanel, also deny direct access to the storage directory:

```nginx
location ^~ /deployee/storage/ {
    deny all;
    return 403;
}
```

For production, the safest option is to restrict `/deployee/` by IP or CloudPanel basic authentication in addition to WordPress authentication.
